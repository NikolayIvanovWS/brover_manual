Архитектура ROS 2 позволяет объединять несколько устройств в одну сеть. В этом режиме все устройства видят общие топики и могут:
- читать данные (например, видео с камеры или одометрию),
- публиковать команды (например, в cmd_vel для управления роботом),
- запускать собственные ноды, взаимодействующие с роботом.

Вы можете объединить настольный компьютер и робот в одну ROS-сеть и работать с ровером удалённо.
:::note
В этой инструкции предполагается, что на вашем компьютере установлены:
- **Ubuntu 24.04** (или новее)
- **ROS 2 Jazzy**
Официальная инструкция по установке ROS 2 Jazzy:
https://docs.ros.org/en/jazzy/Installation.html
:::

### Подключение через FastDDS (Discovery Server)

**Рекомендуемый способ подключения** — использование `FastDDS` в режиме [Discovery-Server](https://docs.ros.org/en/jazzy/Tutorials/Advanced/Discovery-Server/Discovery-Server.html).

Робот настроен на работу в этом режиме. Он особенно удобен:
- при работе по Wi-Fi,
- если в сети несколько роботов,
- если требуется стабильное обнаружение нод.

В этом режиме вы подключаетесь к роверу по его IP-адресу и:
- видите все его топики,
- можете публиковать в них данные,
- можете запускать свои ноды на ноутбуке.

## Fast DDS — режим Discovery Server через переменные окружения

По умолчанию робот работает с *middleware Fast DDS* в режиме `Discovery Server`.
В этом режиме **ровер выступает как сервер** обнаружения узлов, а **компьютер подключается к нему как клиент**.


### 1.1 Настройка на компьютере

Откройте терминал и выполните:
```
export ROS_DISCOVERY_SERVER="192.168.1.41:11811"
export ROS_SUPER_CLIENT=TRUE
```
где:

`192.168.1.41` — IP-адрес ровера, вам необходимо указать **ip адрес своего ровера**;

`11811` — порт Discovery Server (по умолчанию).

Назначение переменных

`ROS_DISCOVERY_SERVER` — указывает адрес сервера обнаружения;

`ROS_SUPER_CLIENT=TRUE` — включает режим полного клиента (доступ ко всему графу ROS 2).

### 1.2 Проверка подключения

Выполните:
```
ros2 topic list
```
Если подключение выполнено корректно, отобразится список топиков робота.

Если список топиков не виден или отображается устаревший список, выполните перезапуск daemon:
```
ros2 daemon stop
ros2 daemon start
```
### 1.3 Постоянное применение настроек*

Переменные окружения действуют только в текущем терминале.

Для автоматического применения во всех новых терминалах добавьте строки в файл `~/.bashrc`

Пример:
```
echo 'export ROS_DISCOVERY_SERVER="192.168.1.41:11811"' >> ~/.bashrc
echo 'export ROS_SUPER_CLIENT=TRUE' >> ~/.bashrc
```
После этого выполните:
```
source ~/.bashrc
```
## Подключение через XML-конфигурацию Fast DDS
Данный способ немного сложнее и предназначен для более продвинутых пользователей. 

В этом случае используется XML-профиль Fast DDS.
1. **Где находятся настройки на роботе**

На роботе используется файл:
```
/home/pi/.ros/fastdds_supeclient.xml
```
Чтобы подключиться к ROS ровера, вам нужно использовать аналогичный файл на своём компьютере.

:::tip
#### Далее работаем на компьютере!
:::
2. **Скачивание файла настроек на ноутбук**

Скачайте готовый конфигурационный файл на свой компьютер в корневую папку:
```
cd ~
wget https://raw.githubusercontent.com/voltbro/turtlebro2/master/extra/fastdds_supeclient.xml
```
3. **Изменение IP-адреса**

Откройте скачанный файл (на компьютере):
```
sudo nano fastdds_supeclient.xml
```
Найдите строку:
```
<address>127.0.0.1</address>
```
И замените `127.0.0.1` на **IP-адрес вашего робота**, например:
```
<address>192.168.1.41</address>
```
Сохраните и закройте файл.

4. **Применение настроек**

Экспортируйте путь к файлу:
```
cd ~
export FASTRTPS_DEFAULT_PROFILES_FILE=$(pwd)/fastdds_supeclient.xml
```
Рекомендуется указывать полный путь к файлу.
В данном случае $(pwd) автоматически подставит путь к вашей домашней директории.

5. **Проверка подключения**

В терминале вашего компьютера выполните:
```
ros2 topic list
```
Если всё настроено правильно, вы увидите список топиков ровера.

6. **Если топики не отображаются**

Иногда кэш ROS 2 может содержать устаревшие данные. Попробуйте перезапустить daemon:
```
ros2 daemon stop
ros2 daemon start
```

Если вы изменили названия топиков или какие-то топики были удалены, а ноутбук продолжает видеть несуществующие или неактуальные названия  — перезапуск daemon обычно решает проблему.

:::tip
Все команды `export` и `source` действуют **только в рамках текущего терминала**. 
После закрытия терминала их нужно будет вводить заново. Если вы хотите, чтобы переменные применялись автоматически во всех терминалах, добавьте их в файл `~/.bashrc`
:::
