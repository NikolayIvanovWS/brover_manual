### Работа без Discovery Server (Simple Discovery)

Если:
- в сети всего один робот,
- один компьютер,
- Wi-Fi не перегружен,

можно использовать стандартный режим *Simple Discovery*. В этом режиме подключение осуществляется по совпадению значения `ROS_DOMAIN_ID`. Если на роботе и на компьютере одинаковый `ROS_DOMAIN_ID`, они автоматически оказываются в одной ROS-сети.

1. **Настройка робота**

На роботе откройте файл:
```
cd ~
sudo nano .ros_params
```
Найдите строку:
```
export FASTRTPS_DEFAULT_PROFILES_FILE=/home/pi/.ros/fastdds_superclient.xml
```
Если она не закомментирована — закомментируйте её, поставив перед ей `#`. В этом же файле указан параметр `ROS_DOMAIN_ID`.  **Запомните его значение** (по умолчанию — 10). Сохраните файл и перезагрузите робота.


2. **Настройка ноутбука**

В терминале выполните:
```
export ROS_DOMAIN_ID=<NN>
```
где `<NN>` — значение, которое указано на роботе
(по умолчанию это 10).

Проверьте подключение:
```
ros2 topic list
```

:::tip
Все команды `export` и `source` действуют **только в рамках текущего терминала**. 
После закрытия терминала их нужно будет вводить заново. Если вы хотите, чтобы переменные применялись автоматически во всех терминалах, добавьте их в файл `~/.bashrc`
:::

:::info
В режиме *Simple Discovery* учитывается значение переменной окружения `ROS_DOMAIN_ID`. По умолчанию для Raspberry Pi установлено значение **10**
:::
